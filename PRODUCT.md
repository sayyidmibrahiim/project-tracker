# Product

## Register

product

## Users

Single-user Windows desktop app for a DBS deployment/support engineer managing multiple CR projects. User works in an operations context: tracking UAT-to-production readiness, coordinating Outlook/Teams updates, collecting evidence, writing notes, and keeping local project folders disciplined under time pressure.

## Product Purpose

Project Tracker DBS centralizes CR deployment work into one local-first desktop tool. Success means the user can see project state, update operational metadata, navigate folders/files, coordinate communications, and enforce workflow rules without hunting across Explorer, Outlook, Teams, notes, and spreadsheets.

## Brand Personality

Utilitarian, modern, disciplined. The interface should feel like a serious productivity app for banking operations: dense enough for real work, calm enough for daily use, and modern enough to feel fast rather than legacy.

## Anti-references

Avoid decorative SaaS dashboards, playful gradients, neon/terminal novelty, generic AI-generated card grids, marketing-page drama, and raw prototype roughness. Do not make the UI feel like a toy, a website landing page, or an over-styled analytics product.

## Design Principles

1. Keep the workflow visible: state, deadlines, links, notes, and actions should be close to the work item they affect.
2. Prefer disciplined density: compact layouts are good when they reduce context switching and preserve readability.
3. Make risky actions explicit: deletes, state transitions, Outlook/Teams actions, and folder moves need clear feedback and guardrails.
4. Preserve local-first trust: UI should reinforce that filesystem and metadata remain recoverable, predictable, and user-controlled.
5. Modernize without decoration: polish spacing, hierarchy, motion, and consistency; avoid visual noise that does not help the task.

## Accessibility & Inclusion

Target WCAG AA. Maintain strong text contrast, visible keyboard focus, readable compact typography, reduced-motion alternatives for non-essential animation, and consistent semantics for buttons, inputs, dialogs, tables, and status feedback.
