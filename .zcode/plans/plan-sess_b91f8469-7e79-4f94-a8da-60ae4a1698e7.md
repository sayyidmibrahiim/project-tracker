## Root Folder Bootstrap & Appcode First-Run Setup ‚Äî Implementation Plan

**Spec:** `_docs/specs/superpowers/specs/2026-07-11-root-folder-bootstrap-design.md`
**Full plan:** `_docs/specs/superpowers/plans/2026-07-11-root-folder-bootstrap.md`
**Branch:** `general/root-folder-bootstrap` from `main`

### Goal
Force root folder to `Documents\Project Tracker` on every launch (create or migrate), then require at least one appcode via popup before app is usable.

### Architecture
- **Backend:** `bootstrap_root()` runs in `app_web.run()` before `webview.start()`. Creates default root, migrates old data if needed (shutil.move + path rewriting in settings.json, appcode.json, SQLite cache), rebuilds cache.
- **Frontend:** Replace `FirstRunSetup.svelte` with `AppcodeSetup.svelte` ‚Äî modal popup calling existing `appcode_add` bridge. [Done] disabled until 1+ appcode saved. No skip.

### Path Audit (what gets rewritten during migration)
- **settings.json:** root_folder, second_brain_folder, file_template_folder, email.template_folder_path, teams.automations[*].attachment_paths
- **appcode.json:** cicd_shared_path (only if shared_root mode + inside old root)
- **SQLite (non-rebuildable, SQL REPLACE):** notifications.project_path, approval_polling_jobs.project_path
- **SQLite (rebuildable, DELETE + rescan):** project_index, drone_tickets, scan_warnings
- **Safe (no action):** project_data.json, link_bank.json, app logs

### Existing behavior reused (no new code)
- `appcode_add` bridge already creates full D-0008 worktree: `appcode.json`, `CICD/`, `{year}/CR/{5 states}/Non-CR/`. Popup calls this directly.
- `ScannerService.rebuild_year()` for cache rescan after migration.

### Edge cases handled
- Root folder `None` ‚Üí create default
- Root already at default ‚Üí no-op
- Old root missing ‚Üí create empty default, clear cache
- Migration failure (disk full) ‚Üí keep old root, log error, app starts normally
- Default root already has data (reinstall) ‚Üí no-op, appcode check runs normally
